Saturday, 4. November 2006

C4J Plugin Runtime 2.5.2.1 released

This is a bugfix release, resolving the following issue:

  • Leading slash in c4j_library variable on Windows [1590461]

If you are updating from a previous version, you may need to reenable C4J capabilities for your project (i.e. removing and enabling C4J again in the C4J Tools menu) after the update.

Friday, 3. November 2006

C4J Plugin Runtime 2.5.2.0 released

This is a synchronization release of the runtime plug-in. It contains now version 2.5.2 of the c4j.jar library.

Monday, 18. September 2006

C4J Plugin 0.1.1 released

Although the news entry for the initial 0.1.0 release is missing, it happened about 2 weeks ago. Now there is a bugfix release, which solves the following problems:

  • All problem markers are reported as errors [1560266]
  • NPE when changing VM args of SWT launch configurations [1558901]
  • contractClass annotation attribute blocks plugin [1558900]
  • NPE when starting Eclipse after a fresh install of the plugin [1552981]

If you are using the Update Site to install new releases, you will have to choose "Search for new features to install", instead of "Search for updates of the currently installed features" in the Find and Install... Dialog. This should be fixed from now on.

Friday, 1. September 2006

Direct Download Problem resolved

There has been a problem with the direct download option for the runtime plug-in. The file is now a .zip and contains the right directory structure so it can be unzipped right under the Eclipse installation directory.

Sorry for the inconvenience.

Thursday, 31. August 2006

C4J Plugin Runtime released

This is the first release of the C4J Plugin Runtime plug-in (net.sourceforge.c4jplugin.runtime). It wraps the Design by Contract Library from the C4J project and comes with a fair amount of documentation.
